  1. 心路历程

    当你看着一本厚厚的红宝书，不要害怕，背着背着就薄了；当你看到US ranking里面的一排学校，也不要慌张，查着查着就没了；当你开始打开word写PS的时候，不要无助，地里有大把大把的资料和高手；当你开始写第一封陶瓷信的时候，不要怯场，有着大把大把的老师可以候选。不要害怕，每一件事情只要开始了，就会有完成的那一天。所以，决定好了，不要怕，先上手，慢慢调整。你犹豫中度过的每一天，都会在最后发现，自己要是早开始一天，就好了。
    套磁不在于时间早晚，在于质量与契合度。因此，打一场有准备的战，发一封高质量的套磁信，才是最关键的地方

  2. Facebook实习ad-小垃圾的成长辛酸史-负能量爆表慎入

    完全无助的状态。各种压力在一起，当时整个人都是崩溃的感觉。这种心酸现在回想都是切肤的痛楚，不断的充满幻想的刷邮箱，掷硬币；蒙头睡到很晚不起床，只是怕起床后又是要面对各种不会做的作业，看不懂的面试题，每次都不可能一下子bugfree的code。我把那套西服封存了一下，皮鞋也扔到了床下落灰，觉得是它们给我带了的坏运气。 整个人极度发胖，脸上长痘痘，干燥，崩溃！身边的不少人刺此刻都拿到了google facebook的ad。
    可以该面对是还是要面对，那时候我已经一周多没做题了，什么都做不进去，大脑一片浆糊，自己的心理调整能力太差，无法克制焦虑，大脑single thread，一直block在当前焦虑进程中，无法做别的事情，多希望time-sharing啊!
    第二天坐着cab到了9号楼，竟然是第一个到的，跟着参观一下大楼，就去参加onsite，是0 1 2 排序的问题，完全不会（就在leetcode，但是没做到，就在10道题没做的里面）。我用了40分钟狂讨论，狂想，狂改各种方案，面试官无奈叹息，最后竟然在42分钟想出来，写出code，（有bug，被面试官ignore）。面试官之前说要给两道题做，可是我已经没有时间了，当时我真是悲愤心中生，只盼钻地缝，然后我几乎是喊着说：快说第二题吧，还有3分钟。
    他摇了摇头，说道：不了，我在乎的是你解决问题的过程，不在于多少道题。我当时知道我完了。

  3. 经历7个月 收到Google Offer

    2014年10月是找工的好季节，虽然我们开学才2个月，周围有规划的人都已经开始好好找工作了，但是我却觉得 课业那么忙 还没准备好呢，我要好好准备，明年再申请，一定能找到好工作的。（不知道有没有类似想法的同学）
    在2015年初大家都在积极认真的找工作的时候，楼主刷了1题leetcode，抱着侥幸的态度就跑去面试了Amazon, 然后被拒了。那个时候反思了一下，也认识到自己的确不够强，所以想：嗯 还是留着大公司先不投，等准备好了再一举拿下吧！
    2015年5月，毕业的时候 一半的同学都已经拿大大小公司的Offer了，那时候还沉浸在毕业的气氛当中，觉得突然时间过的好快，自己已经不是学生了(9个月的master就是这样)，顿时感觉周围同学都已经开始向前走了。这时候才意识到找工作迫在眉睫，所以在5-7月，2个月的时间里刷了1遍leetcode, 花了2个月 其实也是边玩边刷。
    2015年7月，我从东岸搬到了湾区，准备破釜沉舟，在此地把大公司Offer拿下。
    回想了下这半年来的自己，感悟颇多，总觉得自己没能把career这件事重视起来以致如此，
    找工作的时间长了，人难免会难过伤心，但是希望大家看了我的例子之后知道你并不是一个人，这时候要好好保持良好的心态和信心，积极准备，不要过度颓废和失落。我很理解你们的心情，因为我也曾经那样过。
    能尽早准备的尽量尽早，不要总觉得我题都没刷完就要开始找，还是等等吧。有位好朋友跟我说过：人从来都不是在完全准备好的情况下做一件事的，你永远不可能做到100%的准备而去信心满满的面试，把这当做一件挑战和经验，也是一件很有趣的事。
    大家也都知道要刷leetcode,我就不强调了，但是刷的方法我还是提一下, 我自己在刷每一遍的时候，如果碰到没思路或者不顺的题目我会记录下来，在跑过之后我都会记录一条总结方法或者心得，每一遍我都整理成一个note, 一共有3篇。-google 1point3acres
    cracking the code interview, 是我在刷完3遍之后看的，感觉有点小用，加深了自己对各种的理解。主要我是看章节，题目略看，里面有用的也整理成了一个笔记。
    geeksforgeeks，这是一个特别特别特别特别好的网站，重点推荐，刷完题之后刷这个，理解了之后你完全可以 handle 任何大公司的面试。如果说在interview里面各种资料有用度评分的话 leetcode我给30分，geeks我给60分。我在平时碰到不了解的algorithm或者data structure的时候都会去参考，里面很详细。到了11月份我索性把他里面所有的algorithm和data structure都刷了一遍，收益太多。另外geeks也有个前辈整理的手机app, Apple store叫做Geek Reader，这样可以在平时出去无聊的时候看看里面的小章节。
    system design 地里也有一些资料的推荐，可以参考，其实这个你花最多2-3天就可以掌握了，可以侃侃而谈，何乐而不为。我记得在G Onsite最后一轮，即使是不要求的，我还是和他谈了system design里面的很多概念，让他很满意。（想想我也应该是3，4轮被强推才有机会），所以各位，如果你愿意花1-2天的时间，就去准备吧, 当然这个东西 G 家明确要求new grad不需要掌握，而Fb很看重！我也整理成了一个笔记

  4. 发点牢骚，不要太计较一时的得失

    我们中国的孩子呢，用功，自尊心强，这是好处。但是有时候就是太赶节奏了，人生不是赶鸭子上架，也不是dota上高拿塔，就算你耽误一两年，就算第一份工作如屎一样，对你一生来说，也真不算太大的事情。
    学习是一个长期的事情。就算你现在没刷题，运气好进了好公司，你一样要学习，而且你以前学的越少，你现在就要学的越多，否则performance不好，不是很容易被裁掉吗。两三年以后，或者你升不上去不给办绿卡之类的，或者你要跳槽，到时候一样要刷题。
    只要安安静静的做事情，你只需要一个机会，找工作你只要抓住了一个机会，就行了。但是这个机会并不是你在这哭，郁闷，难过就会出现的，你能做的只有等待，并且安静的做事情
    应试教育给了我们太多的攀比心理，现在其实不需要比了，零和思维是最可怕的，我们要的是双赢。和身边的朋友多交流，不管是学习上还是生活上，多排解抑郁和苦难，分享快乐和悲伤，在平淡的生活中创造点不平淡的难忘的日子。就是这样，唠叨完了，最后一句是，胖子不是一口吃成的，是一口一口吃成的

找工作前景？

  1. CS硕士找工作总结……和一些经验

    刷Leetcode，Geeksforgeeks，算法题设计题智力题
    CMU的TOC对找工作帮助很大
    startup的面试普遍比大公司难，面试官会问很多他自己都不会解的问题
    平时写代码（无论是刷题还是写作业）要注重bug-free. 很多时候总想着等编译出错了回头改就好了，面试会认真起来的。但是实际上面试的时候人很紧张很难考虑全面，如果有bug回头再改，就很浪费时间很伤士气了
    简历很重要——虽然都说简历只是敲门砖，但是敲门砖本身就是很必要的。而且hiring manager做决定的时候感觉还是要参考简历的。每隔几天就要拿出来改一下，有时候一天要改好几下，版本号从1.0刷到了目前的10.3……
    cc150 careerup
    口语超级重要……很多题（特别是设计题）面试官是不期待你能一下子做出来的，这期间你的反馈和交流就尤其重要了。如果能和面试官做到谈笑风生的话，即使题没做出来对方对你的评价也不会太差的。
    Linkedin 上的profile还是挺重要的。好好打理一下就会有hr找上门。而且一个月的会员免费试用期也要利用起来，inMail和看“谁查看了我的页面”是两件求面试的神器。
    面试很重要的一点就是要强行自信，装成屌屌的样子面试官也会误以为你屌屌的
    如果想fall学期搞定工作的话，正好暑假刷题，8月份开始就可以投简历+热身了，9,10月份集中面试。

  2. CS找工作心得 从不知道什么是数据结构到找到年薪十万的工作

    headfirst java, head first 系列， datastructure in java 把所有数据结构用你熟悉的语言实现的书就好了，自己从头到尾把所有数据结构实现几遍，知道哪些地方容易弄错，比如链表头尾节点处理之类的多看一下。然后认认真真的看一下相关的算法 Crackingthe coding interview,正儿八经开始刷题了，每个题都要自己写出来。大家一直强调手写代码，其实在电脑上打代码也挺重要的，所以有时间的话手写然后用电脑写，然后比较差别。之所以说电脑打代码也挺重要因为前面一两轮应该是phoneinterview,很多时候也要在电脑上面打代码，所以如果电脑上面代码速度快质量高的话，非常impressive。 有好几个interviewer说我勒个去，你打代码这么快啊。。。。。
    面试到后来慢慢的就有了信心，所以现在没信心的朋友不要担心，信心不是一下子马上就有的，而且一点一点累积起来的，之前面试一旦有自己不知道的就觉得很紧张很沮丧，可是现在如果有自己不懂的或者没想到的，就会很心平气和的和面试官讨论，心态非常好，经常面试的大家都开怀大笑，一路都很开心。。。而且变得越来越humble，心态越来越不浮躁。
    面试最重要的是表现出自己的热情。我自已有一段关于光荣与梦想的激情演讲稿，每次面试的时候一有机会就讲出来。后来发现只要讲过这段话的面试基本都会过，即使技术问题没有很完美，甚至有的时候根本就不对，比如面试Bloomberg的时候，一个题目完全没思路，给我提示了也没思路，然后又提示了还是不知道，后来面试官受不了了直接说我把思路告诉你你实现一下吧，结果又没实现。。。可是后来居然通过了。。。因为当时我就讲了这段话，明显感觉到他被impress了。这段话主要就是讲自己多么多么有激情，多么多么希望有机会可以展示才华。希望大家可以意会。。。
    另外就是电话面试的时候实在不行的话可以google一下问题，一般都能找到一模一样的题目，然后简单看一下思路，有了思路就好办多了。后来我的电话面试全都能通过，就是因为偶尔有时候实在是卡住了就立马google一下，然后花几秒钟大概看看就好了。其实这不是好办法，而且容易有依赖性，可是毕竟机会来之不易，能够争取到机会，在Onsite之前好好准备，也是好事，不过真的要慎用哈，关键在于认真准备
    http://join.hired.com/x/kmfEzQ
    这个网站注册之后会有一个test，通过test之后他们会帮你修改你的页面，你的简历，然后让你选一个时间进入一个为期一周的auction
    在这一周里面会有不同的公司来bid你，你可以直接看到他们开出的offer，而且最后如果通过这个网站接受工作的话还可以拿到2000 bonus。
    上面这个连接是我的refer连接，大家通过这个连接注册如果接受offer的话他们也可以给我1337 bonus，不过最重要的是他们说通过refer的话会被优先考虑。
    这个我刚开始觉得是bullshit,不过后来觉得应该是真的
    因为我刚开始auction的时候，前面一天就只有两个公司来看了我的页面，一个bid都没有，心里那个凉啊
    以为一注册会有好多好多offer 然后发愁选那个呢结果发现和小时候愁去北大还是清华一样是白费力气啊
    然后第二天也只有两三个公司来看。。。。。
    到了第三天，刚开始也就只有两三个公司，心灰意冷的过了一天，结果到下午快6点的时候收到了我第一个bid，是个纽约的financial firm，package是十万年薪，加一点bonus，当时吓尿了，毕竟在东边这个价格还是挺高的啊，然后哆嗦着confirm了要面试 当时真的还蛮开心的
    结果第四天开始每天来看的公司数量是几何级的增长，然后收到的bid也一下子多了好多。
    第四天至少就有十个公司来看我的页面，然后收到两个bid，一个在纽约一个在加州。 第五天也至少有十个公司来看，收到twitter, Bloomberg, gilt,还有德州一家，纽约一家的bid
    所以我觉得refer会有用就是因为刚开始根本不太有人注意到我的感觉，可能是因为没人refer又没offer然后排名很低，后来收到一个offer之后开始有人注意然后慢慢的多收到offer就越来越高的排名，所以一开始如果有人refer的话说不定第一天就可以收到不错的offer 不至于等到后面两天才开始大丰收。。。。。
